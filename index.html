KB Article: KB0011253


Successful Login via Windows:

For remote users, resetting the AD password will not allow them to log into Windows. If a remote user is unable to login to Windows (not VPN), DO NOT RESET THE AD ACCOUNT PASSWORD. Have the user try logging in using the most resent password or previous used password. If the current or previous used password do not work, login as the administrator complete the "Re-Joining to Domain - Windows Network Account Login Issue" document.

ID: .\administrator

PW: PW4avonAdmin!

OR, PW: PW4avonAdmin

After logging in as the Admin, have user connect to VPN and login using the user's most recently used password. If the most recently password do not work, try the most previously used password. If the passwords do not allow the user to successfully login to VPN, remote into the machine and troubleshoot the following:

1. Check if the user is typing the correct ID and password

2. Check AD account for a lock out (please see below)

3. Check AD account for an account expire (please see below)

4. Check for Internet connection-Verify the end-user is successfully connect to the Internet, by going to outside websites. If the user is unable to connect to the Internet, have user cycle-power their Internet connection and reboot. If they continue to have Internet connection issues, have the user contact their ISP for Internet relation connection assistance.

After completing the troubleshooting, you can then reset the AD password for a remote user. However verify that you have completed the following:

1. You have remote into the machine

2. AD account is not locked

3. AD account has not expired

4.There is not an Internet connection issue

Successful Login via VPN:

Note: If a user is unable to login to VPN, DO NOT RESET THE AD ACCOUNT PASSWORD. Have the user try logging in using the most resent password or previous used password. If the current or previous used password do not work, remote into machine and troubleshoot the following:

1. Check if the user is typing the correct ID and password

2. Check AD account for a lock out

3. Check AD account for an account expire

4. Check for Internet connection - Vrify the end-user is successfully connect to the Internet, by going to outside websites. If the user is unable to connect to the Internet, have user cycle-power their Internet connection and reboot. If they continue to have Internet connection issues, have the user contact their ISP for Internet relation connection assistance.

After completing the troubleshooting, you can then reset the AD password for a remote user. However verify that you have completed the following:

1. You have remote into the machine

2. AD account is not locked

3. AD account has not expired

4.There is not an Internet connection issue

Reset the AD password (ONLY IF USER IS ALREADY LOGGED INTO WINDOWS). Have the user login to VPN using the new password. Once contacted successfully, lock the computer (DO NOT REBOOT).

*If you reboot the machine, the user will be unable to login to Windows and recover the profile successfully. The profile of the end-user is a cached profile and will be lost if you reset the password and reboot.

After resetting the password and the user have successfully logged into VPN, verify the user successfully connected to VPN and have the user lock and unlock the computer(NOT REBOOT THE COMPUTER).

Once the computer has been locked, while contacted to VPN, have user login to Windows with the new VPN/AD password.

Once logged into Windows successfully and still contacted to VPN, have user login to other applications needed and verify a successful login and connection.

Resetting AD Account Password for Network/In-House Users:

1. Search for the end-user's name or AD ID

2. Right-click on the ID and select 'Reset Password'

Type in a new temporary password and check the 'User mush change password at next login' and click ok.

How to Unlock AD Account:

Select the 'Unlock Account' box and click 'Ok'. You can also use this step as a troubleshooting step.

AD Account Disabled Process:

If an end-user's AD account is showing 'Disabled', DO NOT ENABLE the account. In order to enable the account we need their manager's approval. Please complete the following:

1. Open an INC ticket

2. Verify the end-user's manager name, which the manager's name can be found in AD, under the 'Organization' tab.

3. Email the manager for their approval of re-enabling the AD account. 4. Attach the approval to the INC ticket and assign the ticket to the IBM-IN-IAM.

How to Check if AD Account has Expired:

If an end-user's AD account, 'Account Expires' section has 'End of:' date showing a date that will expire soon or has already expired, DO NOT CHANGE.In order to extend the account we need their manager's approval. Please complete the following:

1. Open an INC ticket

2. Verify the end-user's manager name, which the manager's name can be found in AD, under the 'Organization' tab.

3. Email the manager for their approval to extend the AD account for another 90-days.

4. Attach the approval to the INC ticket and assign the ticket to the IBM-IN-IAM.
